Criação e análise de um modelo de boas práticas para startups de software
Descrição: Startups de software possuem grande poder de empregabilidade, mas as taxas de insucesso deste tipo de empreendimento são cada vez maiores. Dentre os fatores que auxiliam neste contexto está a falta adoção de modelos de boas práticas para o desenvolvimento deste tipo de software, o que aumenta o retrabalho e contribui para o aumento de custos deste projeto. Alguns modelos propostos em engenharia de software são usados neste contexto, no entanto as particularidades de startups de software não são levadas em consideração. Este trabalho visa responder a seguinte pergunta de pesquisa: Como podem ser criados modelos de boas práticas para startups de software focado em requisitos, arquitetura e testes? Os resultados desta pesquisa poderão ser usados em startups de software, além de contribuir para atingir os Objetivos de Desenvolvimento Sustentáveis Indústria, inovação e infraestrutura e Consumo e produção responsáveis.

Definição de critérios para a medição de débito técnico em startups de software
Descrição: Startups de software são extensamente usadas de forma inovativa e contribuem para a inovação de emprego. No entanto, durante as fases do desenvolvimento da startup, não são usados métodos de desenvolvimento de software, o que aumenta o custo de retrabalho no futuro, isto é, o débito técnico em startups de software. Com isso, a possibilidade de insucesso da startup aumenta consideravelmente, uma vez que a evolução do software é prejudicada neste contexto. Diante disso, surge a seguinte questão de pesquisa: Como adaptar uma ferramenta existente para medição de débito técnico em startups de software? Este trabalho visa adaptar os critérios objetivos de débito técnico e ferramentas existentes de medição em engenharia de software para o contexto de startups de software. Com estas medições, será possível monitorar a qualidade do software desenvolvido e aumentar as probabilidades de sucesso em startups de software. Os resultados desta pesquisa poderão ser usados em startups de software, além de contribuir para atingir os Objetivos de Desenvolvimento Sustentáveis Indústria, inovação e infraestrutura e Consumo e produção responsáveis..

2021 - 2022
Um processo de testes de software para projetos acadêmicos
Descrição: Os testes de software fazem parte do campo da linha de pesquisa Engenharia de Software, uma parte da subárea de pesquisa Metodologia e Técnicas de Computação. Eles são usados para aumentar a qualidade do software e diminuir os custos em seu projeto, contribuindo para a satisfação do usuário final. Testes ágeis permitem a adoção de boas práticas para a entrega do software em versões incrementais e testadas conforme estabelecido no início do projeto, sendo adequados a projetos com escopo instável. Projetos de software acadêmicos podem ser beneficiados neste contexto porque projetos de pesquisa frequentemente buscam o ineditismo, o que gera a incerteza do escopo estabelecido inicialmente no projeto. Aliado a isso, projetos acadêmicos continuam sendo executados no contexto da pandemia do COVID-19, o que gerou restrições de isolamento, adoção de trabalho remoto e restrições de custos. Descrição: Sistemas de tempo real são usados em monitoramento de pacientes, celulares e sistemas embarcados. Eles necessitam de testes que incluem o fator tempo, requerendo maior rigor nesta fase.
